# SoloMoney Avito OS

Личный AI-агент для арбитража товаров на Avito. Мониторит твои сохранённые поиски,
прогоняет новые объявления через Gemini, отсекает мусор и скам, и присылает в
Telegram только выгодные варианты с готовым решением: **брать срочно / торговаться /
наблюдать / пропустить / высокий риск** — плюс готовое сообщение продавцу.

> Фокус-категории: **PlayStation 5**, **MacBook Air M1/M2**, **iPhone 13/14/15**.
> Это личная система, не SaaS. Бот работает только для тебя.

---

## Что внутри

| Слой | Технология | Файлы |
|------|-----------|-------|
| Мониторинг Avito | Playwright (без обхода защит) | `app/watcher/` |
| AI-анализ | Gemini, structured output | `app/ai/` |
| Оценка/скоринг | детерминированные правила + прайсбук | `app/core/scoring.py`, `pricebook.py` |
| Сделки/БД | SQLite + SQLAlchemy | `app/models.py`, `app/core/deals.py` |
| Бот | aiogram 3 | `app/bot/` |
| Дашборд (Mini App) | FastAPI + ванильный JS | `app/web/` |

**Двухслойная оценка.** Gemini даёт совещательный вывод, но финальный вердикт,
цифры и риск пересчитывает `scoring.py` по жёстким правилам и сверяет с прайсбуком —
поэтому система не «ведётся» на красивые слова модели, и все решения проверяемы
(журнал в таблице `event_log`).

**Никаких серых схем.** Капчу не обходим: при блокировке агент ставит мониторинг
на паузу, шлёт уведомление и ждёт, пока ты пройдёшь проверку вручную.

## Установка

```bash
# 1) Зависимости
pip install -r requirements.txt
playwright install chromium

# 2) Конфиг
copy .env.example .env        # Windows
# cp .env.example .env        # macOS/Linux
```

Заполни `.env`:
- `BOT_TOKEN` — у [@BotFather](https://t.me/BotFather)
- `OWNER_TELEGRAM_ID` — твой id у [@userinfobot](https://t.me/userinfobot)
- `GEMINI_API_KEY` — в [Google AI Studio](https://aistudio.google.com/app/apikey)

```bash
# 3) Запуск (бот + дашборд + мониторинг)
python run.py
```

Дашборд: <http://localhost:8000>. Бот ответит на `/start`.

> Без `BOT_TOKEN` запустится только веб-дашборд — удобно для первого знакомства.

## Как пользоваться

### Ручной анализ (самый быстрый старт)
Пришли боту текст объявления (заголовок + цена + описание) — получишь карточку:

```
🟢 БРАТЬ СРОЧНО          🔥 Hotness: 34.2

Товар: PS5 Slim Digital
Цена продавца: 30 000 ₽
Рыночная цена: 40 000 ₽
Быстрая перепродажа: 36 000 ₽
Цель покупки: 28 000 ₽
Потенциальная прибыль: 8 000 ₽
Маржа: 28%
Ликвидность: высокая (~3 дн.)
Риск: 15/100
Почему: цена заметно ниже рынка, ходовая модель…
Что проверить: • часы наработки • гарантия/чек • комплект
Что написать продавцу: «Здравствуйте! …»
Решение: 🟢 БРАТЬ СРОЧНО
```

Кнопки под карточкой: сообщения для торга (3 тона), «в сделки», «наблюдать»,
«пропустить», «куплено → черновик перепродажи».

### Авто-мониторинг
1. Зайди на Avito, настрой фильтры поиска (город, цена, состояние), отсортируй
   **по дате**, скопируй URL из адресной строки.
2. Вставь URL в `data/searches.json`, поставь `"enabled": true`.
3. Перезапусти `python run.py`. Агент будет проверять поиски каждые 3–5 минут и
   присылать только варианты с вердиктом «брать» или «торговаться».

Управление: `/watch` (статус), `/pause`, `/resume`, `/searches`.

### Команды бота
`/today` — топ активных сделок · `/deals` — все активные · `/stats` — заработок,
ROI, win-rate, воронка · `/searches` — поиски (вкл/выкл кнопками) ·
`/addsearch URL` — добавить поиск · `/price ID buy|sell|cost СУММА` — записать
фактические цифры · `/calibrate` — подстроить прайсбук под твои реальные продажи ·
`/watch /pause /resume` — мониторинг.

### Деньги: чистая прибыль и учёт
Система считает **чистую** прибыль — из валовой вычитаются расходы (дорога,
комиссия, мелкий ремонт; базовая ставка `DEFAULT_COST_RUB` + надбавка по категории).
Пороги отсева (`MIN_PROFIT_RUB`, `MIN_MARGIN_PCT`) применяются к чистой прибыли,
поэтому в чат не попадают сделки, которые «в плюсе» только на бумаге.

После покупки/продажи фиксируй фактические цифры (`/price` или промпт в дашборде) —
тогда `/stats` показывает реальный ROI, win-rate, средний срок продажи, заработок
за неделю/месяц и сколько денег заморожено в непроданном товаре.

### Напоминания
Агент сам напомнит: выгодный лот висит, а ты не написал; продавец молчит больше
суток; куплено, но не выставлено; товар продаётся дольше прогноза → пора снизить
цену. Не чаще одного напоминания на сделку в ~20 часов.

### Самокалибровка цен
Чем больше закрытых сделок — тем точнее прайсбук. `/calibrate` берёт медиану твоих
фактических продаж по модели и предлагает обновить опорные цены под твой регион и
реальный рынок (применяется по подтверждению).

### Свой каталог товаров
Система не ограничена PS5/MacBook/iPhone — добавляй любые товары, и они сразу
участвуют в парсинге и анализе (категории распознаются динамически).

- `/products` — посмотреть каталог
- `/addproduct категория | модель | рынок | быстрая | high/medium/low`
  — напр. `/addproduct airpods | AirPods Pro 2 | 18000 | 16000 | high`
- `/delproduct модель` — удалить
- В дашборде — секция «Каталог товаров» с формой добавления

Под каждый товар создай поиск (`/addsearch URL`), чтобы мониторинг его отслеживал.
Опорные цены можно править и вручную в `data/pricebook.json`. Чем точнее цифры,
тем точнее вердикты (Gemini опирается на них, scoring сверяет).

### Подбор по бюджету (что брать выгодно)
`/find 50000` — агент пройдёт по каталогу и покажет, что выгоднее всего брать на
эту сумму: ожидаемая цена покупки и продажи, чистая прибыль за флип, маржа,
ликвидность, сколько штук влезает в бюджет и суммарный потенциал. Ранжировано по
марже × ликвидность. Позиции, где маржа уже уточнена твоими реальными продажами,
помечены ⭐️. В дашборде — блок «Что выгодно брать на бюджет».

> Оценка покупки = рыночная цена минус типичная скидка выгодного объявления
> (`OPPORTUNITY_DISCOUNT_PCT`, по умолчанию 22%). Это ориентир направления, а не
> гарантия конкретной сделки — финальное решение всегда по реальному объявлению.

## Деплой на Replit

1. Импортируй проект, добавь секреты (`BOT_TOKEN`, `OWNER_TELEGRAM_ID`,
   `GEMINI_API_KEY`) в **Secrets**.
2. `replit.nix` уже подключает системный Chromium для Playwright.
3. Жми **Run**. Для постоянной работы используй **Deployments / Reserved VM**
   (иначе Replit усыпляет процесс); `GET /health` помогает keep-alive.

**Если Avito блокирует IP Replit** (датацентровые адреса банятся чаще) —
запусти `python run.py` локально на своём ПК. База (`data/solomoney.db`) и весь
функционал те же. На будущее в `app/core/sources.py` заложена абстракция источника:
можно подключить легальный платный парсер-API одним адаптером.

## Безопасность
- Секреты только в `.env` (в репозиторий не попадает, см. `.gitignore`).
- Бот отвечает только владельцу (`OWNER_TELEGRAM_ID`); API проверяет подпись
  Telegram `initData`.
- Никакого обхода капчи, прокси-ротации, авто-рассылки. Сообщения продавцам ты
  отправляешь сам.
- Бэкап БД — кнопка в дашборде или `POST /api/backup` (важно на эфемерном диске Replit).

## Структура
```
app/
├── ai/        # gemini, prompts, analyzer (structured output)
├── core/      # pricebook, scoring, deals, reminders, calibration, sources
├── watcher/   # avito (Playwright), monitor (цикл+бэкофф), searches
├── bot/       # handlers, keyboards, formatting, main
└── web/       # api (FastAPI), security (initData), static (Mini App)
data/          # pricebook.json, searches.json, solomoney.db, backups/
run.py         # бот + дашборд + мониторинг + напоминания в одном процессе
```

> БД мигрируется автоматически: при запуске недостающие столбцы добавляются, старые
> данные не теряются.
